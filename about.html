How the Login Page Was Built

A detailed breakdown of every HTML element, CSS property, and JavaScript concept used to construct the VIT Student Portal login page.

HTML Structure

DOCTYPE & meta tags

<!DOCTYPE html> declares the document as HTML5. <meta charset="UTF-8"> sets character encoding and <meta name="viewport"> enables responsive scaling on mobile devices.

<header> element

The sticky navigation bar uses a semantic <header> tag with an <img> for the VIT logo and a <span> for the portal name. Flexbox positions them side by side.

<input> elements

type="text" for the username field and type="password" for the password field. The onkeydown attribute captures the Enter key to trigger login. placeholder provides hint text inside each field.

<button> element

The Sign In button uses onclick, onmouseover, and onmouseout inline event attributes. The id="loginBtn" attribute allows JavaScript to access and disable it via the DOM.

Semantic layout tags

<main>, <footer>, and <header> are used for semantic structure. <label> tags with for attributes are linked to each input for accessibility.

CSS Properties Used

CSS Custom Properties (Variables)

All colours are defined in :root using var(--name) syntax — e.g. --sage-dark: #3d5a47. This keeps the theme consistent and easy to change globally.

Flexbox Layout

display: flex, flex-direction: column, align-items: center, and justify-content: center are used throughout to centre the login box and align header elements.

backdrop-filter & opacity

The sticky header uses backdrop-filter: blur(12px) with a semi-transparent background: rgba(247,244,239,0.92) to create a frosted glass effect while scrolling.

Transitions & box-shadow

All interactive elements use transition: background 0.22s, transform 0.18s. The login box uses box-shadow: 0 4px 20px rgba(0,0,0,0.07) for depth. The button lifts with transform: translateY(-2px) on hover.

radial-gradient background

The page background uses layered radial-gradient and linear-gradient on a position: fixed pseudo-element (body::before) to create a soft ambient glow effect without affecting layout.

position: sticky & z-index

The header stays visible during scroll using position: sticky; top: 0 combined with z-index: 100 to always render above page content.

JS

JavaScript Concepts

Variables

var CORRECT_USER, var CORRECT_PASS, var MAX_ATTEMPTS = 3, and var attemptsLeft store the credentials and track remaining login tries.

if / else Conditions

Conditions check whether fields are empty, whether credentials match, and whether attemptsLeft <= 0. Each branch produces a different user-visible outcome (error message, lock, or redirect).

onclick Event Handler

The Sign In button calls attemptLogin() via onclick. The same function is also triggered by the Enter key using onkeydown="if(event.key==='Enter') attemptLogin()" on both inputs.

DOM Methods

document.getElementById() retrieves elements by ID. element.value reads field values. element.textContent writes error messages dynamically. element.className switches CSS classes for styled alert boxes.

Disabling Button via DOM

After 3 failed attempts, document.getElementById("loginBtn").disabled = true prevents further submissions. Both inputs are also disabled via element.disabled = true to lock the entire form.

focus() Method

element.focus() is called on the first invalid field, directing the user's cursor to the problematic input automatically without requiring manual click.

window.location.href (Redirect)

On successful login, window.location.href = "about.html" inside a setTimeout() delay of 1 second navigates the user to this about page after a brief confirmation message.

for Loop & innerHTML

The attempt indicator dots are generated using a for loop that builds an HTML string and sets it via element.innerHTML. Used dots get the class "used" to turn red visually.

✎

Security Logic Summary

The login system enforces a hard limit of 3 attempts. On each failed try, attemptsLeft is decremented and the dot indicators update. When it reaches zero, both the button and inputs are disabled via JavaScript DOM manipulation — no page reload required. Credentials are compared directly using strict equality (===).
